GMGN token holders Top100 单个钱包字段说明
=========================================

来源命令:
gmgn-cli token holders --chain sol --address <CA> --limit 100 --raw

说明:
每个 list item 代表一个当前持有该 token 的钱包或池子地址。Top100 默认按 amount_percentage 从高到低排序。
注意 addr_type=2 通常是交易池/交易所地址，不应该当普通钱包参与庄家/散户成本分析。


一、身份与地址
-------------
address
  钱包地址或池子地址。普通钱包分析主要看这个字段。

account_address
  Token Account 地址，不等同于钱包地址。Solana 上一个钱包可以有多个 token account。

addr_type
  地址类型。0 通常是普通钱包，2 通常是交易池/交易所/LP 地址。
  当前样例 addr_type=2，所以它是 pump_amm 池子，不是普通持有人。

exchange
  如果 addr_type=2，这里会显示池子或交易所名称，例如 pump_amm、raydium。

wallet_tag_v2
  GMGN 的排名标签，例如 TOP1、TOP2。表示在当前 holders 排名里的位置。

name / avatar / twitter_username / twitter_name
  GMGN 识别到的钱包展示名、头像、Twitter 信息。为空说明没有识别身份。


二、当前持仓
-----------
native_balance
  钱包原生币余额。Solana 下通常是 lamports，需要除以 1e9 才是 SOL。
  可用于判断钱包是否空壳，但池子地址不适用。

balance
  当前持有的 token 数量。

amount_cur
  当前持有的 token 数量，通常和 balance 一致。

usd_value
  当前持仓按当前价格折算的美元价值。

amount_percentage
  当前持仓占总供应比例。0.2257 表示 22.57%。

accu_amount
  累计统计口径下的 token 数量。样例中是池子相关微量记录，不适合当普通钱包买入量。


三、成本字段
-----------
cost
  当前成本估算。

cost_cur
  当前持仓成本估算。

accu_cost
  累计成本估算。

total_cost
  总成本。普通钱包可用于计算盈亏基准。

avg_cost
  平均买入成本。普通钱包做成本线分析最重要的字段。
  如果为 null，说明没有正常买入成本数据，可能是池子、转入、或 GMGN 无法计算。

avg_sold
  平均卖出价格。没有卖出时通常为 null。

注意:
成本线分析建议优先使用 avg_cost。
如果 avg_cost 为空，不建议用 cost/accu_cost 替代普通钱包成本，尤其 addr_type=2 的池子地址应排除。


四、买入/卖出汇总
----------------
buy_volume_cur
  当前周期或当前 token 维度累计买入金额，单位 USD。

sell_volume_cur
  当前周期或当前 token 维度累计卖出金额，单位 USD。

buy_amount_cur
  累计买入 token 数量。

sell_amount_cur
  累计卖出 token 数量。

sell_amount_percentage
  已卖出比例。0 表示未卖，1 表示买入部分基本卖完。

buy_tx_count_cur
  买入交易次数。

sell_tx_count_cur
  卖出交易次数。

current_buy_amount
  当前统计窗口内买入 token 数量。

current_sell_amount
  当前统计窗口内卖出 token 数量。

netflow_usd
  GMGN 原始净流字段，通常是卖出收入 - 买入成本。
  注意在 deep_alpha_pro.py 里为了更符合“净买入”直觉，已转换为 buy_volume_cur - sell_volume_cur。

netflow_amount
  token 数量净流，通常是买入数量 - 卖出数量。


五、转账字段
-----------
current_transfer_in_amount
  当前统计口径下转入 token 数量。

current_transfer_out_amount
  当前统计口径下转出 token 数量。

history_transfer_in_amount
  历史累计转入 token 数量。

history_transfer_in_cost
  历史转入成本估算。

history_transfer_out_amount
  历史累计转出 token 数量。

history_transfer_out_income
  历史转出收入估算。

history_transfer_out_fee
  历史转出手续费。

transfer_in_count
  转入次数。

transfer_out_count
  转出次数。

transfer_in
  是否被 GMGN 标记为转入型持仓。

分析用途:
如果一个普通钱包持仓很高，但 buy_volume_cur=0、transfer_in_count>0，说明不是市场买入，而是转入/分发，可能是团队钱包或关联钱包。


六、历史买卖成本与手续费
-----------------------
history_bought_cost
  历史买入总成本，单位 USD。

history_bought_fee
  历史买入手续费。

history_sold_income
  历史卖出收入，单位 USD。

history_sold_fee
  历史卖出手续费。

realized_profit
  已实现利润，通常来自卖出。

realized_pnl
  已实现盈亏比例。

unrealized_profit
  未实现利润，即当前持仓浮盈/浮亏。

unrealized_pnl
  未实现盈亏比例。

profit
  总利润，通常包含已实现 + 未实现。

profit_change
  总盈亏比例。


七、状态与风险标记
----------------
is_new
  GMGN 是否认为这是新钱包。

is_suspicious
  是否被 GMGN 标记为可疑钱包。

is_on_curve
  是否仍在 bonding curve 阶段。

tags
  GMGN 平台级钱包标签，例如 smart_degen、renowned、kol、dex_bot 等。

maker_token_tags
  当前 token 维度的钱包行为标签。
  常见值:
  - top_holder: 大持有人
  - bundler: 捆绑买入
  - rat_trader: 老鼠仓/内幕
  - sniper: 开盘狙击
  - paper_hands: 纸手

当前样例 maker_token_tags=["top_holder"]，表示它是大持有人标签。


八、时间字段
-----------
start_holding_at
  首次持有该 token 的 Unix 时间戳。可用于分析同一时间段进场、同批吸筹。

end_holding_at
  完全退出该 token 的时间。null 表示当前仍持有。

last_active_timestamp
  该钱包最近一次与该 token 相关的活跃时间。

last_block
  最近活跃区块。

created_at
  钱包创建时间。0 表示未知。
  可用于判断新钱包、同批创建钱包、阴谋钱包。


九、最近交易/转账记录
-------------------
native_transfer
  最近一次原生币转账记录，例如 SOL 转账。
  子字段通常包含 name、address、timestamp、tx_hash、type。
  可用于追踪资金来源。

token_transfer
  最近一次 token 相关转账/交易。

token_transfer_in
  最近一次 token 转入来源。
  可用于分析 token 来源聚类。

token_transfer_out
  最近一次 token 转出目标。
  可用于发现出货或分发路径。

字段结构:
name
  地址名称，如果 GMGN 识别到。
address
  来源或目标地址。
timestamp
  时间戳。
tx_hash
  交易哈希。
type
  转账/交易类型。


十、当前样例的关键判断
--------------------
这个样例:
address=22dKrrZ1pXKbmvatRaBqBPyECbM2qvKyjy3eaYZ4Nyfe
addr_type=2
exchange=pump_amm
amount_percentage=0.22574750704941027
usd_value=6594.316286207619
buy_volume_cur=0
sell_volume_cur=0
avg_cost=null
maker_token_tags=["top_holder"]

结论:
这是 pump_amm 池子地址，不是普通钱包。
它持有 22.57% 是池子流动性里的 token 储备，不应该算作庄家持仓、Top10 钱包成本、买卖净流或新钱包分析。
在代码里应继续通过 addr_type=2 或 exchange 包含 pool/amm/raydium/pump_amm 排除。


十一、适合进入分析模型的字段优先级
--------------------------------
普通钱包持仓:
  address, amount_percentage, usd_value, amount_cur

买卖行为:
  buy_volume_cur, sell_volume_cur, buy_amount_cur, sell_amount_cur,
  buy_tx_count_cur, sell_tx_count_cur, sell_amount_percentage

成本线:
  avg_cost, total_cost, unrealized_pnl, profit_change

吸筹/出货:
  holder_net_buy_usd = buy_volume_cur - sell_volume_cur
  sell_amount_percentage
  last_active_timestamp

同源/阴谋钱包:
  native_transfer.address
  token_transfer_in.address
  created_at
  start_holding_at
  maker_token_tags

风险标签:
  tags
  maker_token_tags
  is_new
  is_suspicious
