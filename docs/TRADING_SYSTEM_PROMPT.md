# TRADING SYSTEM PROMPT — Alpha + Bottom 交易系统

> 本文件为 AI 可读的交易策略 prompt，供后续分析和自动化使用。
> 版本: v2.1 | 数据范围: 2026-05-18 ~ 2026-05-21 | 信号源: alpha_push_events + bottom_top100_push_records

---

## 一、策略概览

### 覆盖信号
- Alpha 1m 打新: `alpha_push_events` 表，path_class 字段
- Bottom 底部异动: `bottom_top100_push_records` 表，signal_type 字段
- 总信号池: ~400+ (4天)

### 核心原则
**不在信号触发时立即买入。等价格回撤到信号价的 -35%~-40% 再挂限价单。**

原因: 60%信号走"先跌后涨"路径。直接从信号价买入 = 44%胜率/+27%收益。等回撤入场 = 100%胜率/+130%+收益。

### 回测表现 (均衡型策略)
- 入场: 24笔 (从 300+信号中筛选)
- 胜率: 88-100%
- 均收益: +130%~+271%
- 盲买均收益: +14.3% (对比基准)

## 二、Bottom 信号 7层策略 (含量能确认)

### Layer 1: 信号类型过滤
```
优先级: abnormal > new_revival > drop_40w > quiet_runup
丢弃: quiet_breakout
加分: 同一CA触发2+种信号 → 得分+2
```

### Layer 2: 标的质量过滤
| 指标 | 通过范围 | 甜点区 | 丢弃线 |
|------|---------|--------|--------|
| MCAP | 50K-300K USD | 50K-150K | <30K 或 >500K |
| 年龄 | 1h-24h | 1h-6h | <10min |
| 流动性(pool/mcap) | 15%-30% | 15%-30% | <12% |
| 信号时涨幅 | <500% | — | >500% |

### Layer 3: 量能确认 (Bottom 独有)
```
K线周期: 5m
等待: 信号后 5根K线 (25分钟)

post/pre量比 = 信号后5根K线平均成交量 ÷ 信号前5根K线平均成交量

判断:
  > 1.2  → 放量，真实买盘，正常仓位进入
  0.8-1.2 → 持平，减半仓位(5%)
  < 0.8  → 缩量，死猫反弹，丢弃不交易

历史验证: 胜者组 post/pre=1.32，败者组=0.69
```

### Layer 4: 入场规则
```
限价买单价格 = 信号K线收盘价 × 0.60~0.65 (-35%~-40%回撤)

激进: ×0.65 (-35%)    更多成交
均衡: ×0.60 (-40%)    推荐 ★
保守: ×0.55 (-45%)    最高质量

最大等待: 8小时未成交 → 取消
```

### Layer 5: 风控
```
硬止损: 入场价 × 0.75 (-25%)
提前砍: 没到+50%且跌到-20% → 主动砍
移动止盈激活: 入场价 × 1.50 (+50%)
移动止盈距离: 峰值 × 0.85 (回落15%卖出)
保本保护: 到过+50%后跌回入场价±5% → 保本出
```

### Layer 6: 仓位管理
```
单笔: 5%-10% 总资金
最大并发: 3-5个
日最大亏损: 15% 总资金 → 触发后当日停止
```

### Layer 7: 止盈
```
50%仓位 @ +50% 固定止盈
30%仓位 @ +100% 固定止盈
20%仓位 @ 15%移动止盈 (飘仓)
```

## 三、Alpha 1m 信号策略 (path_class 驱动)

### 信号分类
Alpha 信号来自 `alpha_push_events` 表，`raw_stats->post_push_track->path_class` 字段：

| path_class | 含义 | 策略决策 |
|-----------|------|---------|
| 回撤后上涨 | 先跌后涨，V型反转 | ✅ 进入过滤 |
| 直接上涨 | 信号后直接拉盘 | ❌ 跳过 (93%最终亏损) |
| 直接下跌归零 | 一路下跌 | ❌ 跳过 (100%亏损) |
| 未达20%观察 | 未触发任何分类 | ❌ 跳过 |

### Alpha 三层过滤
```
Layer 1: path_class == "回撤后上涨" → 通过 (淘汰62%)
Layer 2: 最大回撤 >= 40% → 等到了才入场 (淘汰21%)
Layer 3: 最大回撤 < 65% → 不归零才入场 (淘汰11%)
```

### Alpha 入场后逻辑
与 Bottom Layer 5-7 完全相同: -25%硬止损, +50%激活移动止盈, 15%回撤, 保本保护。


## 五、路径分类系统

### Bottom 路径 (基于 time_to_low vs time_to_peak)
```
if time_to_low < time_to_peak:
    if max_gain > 100%   → V_moon     (深V飞天)
    if max_gain > 30%    → V_strong   (强V反转)
    if max_gain > 10%    → V_normal   (一般V)
    else                 → V_weak     (弱V)
else:
    if cur_ret < -20%    → pump_crash (拉高崩盘)
    if max_gain>30% and cur_ret>10% → pump_hold (拉高稳住)
    if max_gain<10% and cur_ret<-5% → dead_floor (躺平)
    else                 → pump_fade  (拉高回落)
```

### 路径分布
| 路径 | 占比 | 平均最大涨幅 | 平均当前收益 | 策略态度 |
|------|------|-------------|-------------|---------|
| V_moon | 11% | +305% | +138% | 首选 |
| V_strong | 17% | +59% | +20% | 首选 |
| V_normal | 8% | +19% | -0.1% | 可接受 |
| V_weak | 4% | +6% | -1% | 谨慎 |
| pump_hold | 2% | +71% | +35% | 可观察 |
| pump_fade | 15% | +29% | -7% | 避免 |
| pump_crash | 32% | +27% | -37% | **严禁** |
| dead_floor | 11% | +4% | -12% | **严禁** |

### Alpha 路径 (来自 post_push_track)
```
回撤后上涨 → 对应 Bottom 的 V_pattern 系列 → 进入过滤
直接上涨   → 对应 Bottom 的 pump_crash → 跳过
直接下跌归零 → 对应 Bottom 的 dead_floor → 跳过
未达20%观察 → 未触发任何分类 → 跳过
```

## 八、执行检查清单

```
收到信号时:
□ 信号类型: abnormal/new_revival/quiet_runup? (Bottom) 或 path_class=="回撤后上涨"? (Alpha)
□ MCAP 50K-300K?
□ 年龄 1h-24h?
□ 流动性 15%-30%?
□ 信号时涨幅 <500%?

等15-25分钟后 (仅Bottom):
□ 5m K线 post/pre量比 >1.2?

确认通过后:
□ 挂限价单 @ 信号价×0.60~0.65
□ 设硬止损 @ 入场价×0.75
□ 设移动止盈激活 @ +50%, 回撤15%
□ 设保本保护
□ 设分批止盈: 50%@+50%, 30%@+100%, 20%移动

成交后:
□ 每天检查持仓
□ 日亏损15% → 当日停止
□ 8小时未成交 → 取消限价单
```

## 十、数据源说明

### Alpha 1m
- 表: `alpha_push_events`
- 路径分类: `raw_stats->post_push_track->path_class`
- 值: "回撤后上涨" / "直接上涨" / "直接下跌归零" / "未达20%观察"
- 关键字段: max_gain_pct, current_return_pct, max_drawdown_pct, entry_price, peak_price

### Bottom 底部异动
- 表: `bottom_top100_push_records`
- 信号类型: abnormal, new_revival, quiet_runup, drop_40w, drop_50w, quiet_breakout
- 性能CSV: `gmgn_outputs/bottom_top100_push_performance_*.csv`
- K线API: `https://dquery.sintral.io/u-kline/v1/k-line/candles`
- K线参数: platform=solana, interval=5min, pm=p

### 样本局限
- 4天数据，样本量有限
- 回测假设完美执行（无滑点、无限价单必然成交）
- 实盘需考虑: 滑点、流动性不足、网络延迟、限价单不成交
