## 策略总览

```
信号推送 → 过过滤器 → 等量能确认 → 挂限价单 → 成交后设风控 → 分批止盈
  ↓           ↓           ↓            ↓            ↓            ↓
 Layer1     Layer2      Layer3       Layer4       Layer5     Layer6+7
```

### 回测表现

| 策略变体 | 交易数 | 胜率 | 平均收益 |
|----------|--------|------|----------|
| 盲买持有（基准） | 150 | 44% | +26.9% |
| DD35%入场+全过滤 | 8 | 100% | +130.5% |
| DD40%入场+全过滤 | 5 | 100% | +138.1% |

### 策略核心逻辑

**不在信号触发时立即买入。等价格跌到信号价的 -35%~-40% 再进场。** 原因是：60%的信号都走"先跌后涨"（V型反转）路径，先跌的这段回撤才是最佳入场点。

## 第一层：信号过滤

### 规则

收到 Telegram/前端推送后，检查信号类型：

| 信号类型 | 优先级 | 说明 | 得分 |
|----------|--------|------|------|
| `abnormal` | 最高 | 异常成交量/价格行为，最可靠的信号 | +3 |
| `new_revival` | 高 | 老币重新活跃（底部复苏） | +3 |
| `drop_40w` | 中 | 从周期高点大幅下跌后的反弹 | +2 |
| `quiet_runup` | 可接受 | 低成交量的稳步上涨 | +1 |
| `quiet_breakout` | 不推荐 | 数据量不足 | 0 |
| 多信号叠加 | 加分 | 同一CA触发2+种信号 | +2 |

### 操作

```
收到推送 → 看 signal_type
  abnormal 或 new_revival → 进入第二层
  quiet_runup → 进入第二层，但降低优先级
  drop_40w/50w → 进入第二层，但数据量小需谨慎
  其他 → 丢弃
```

## 第二层：标的质量

### 规则

逐项检查推送携带的数据：

| 指标 | 范围 | 得分 | 不通过就丢弃 |
|------|------|------|-------------|
| MCAP | **50,000 ~ 300,000 USD** | +2 (50K-150K), +1 (150K-300K) | <30K 或 >500K |
| 年龄 | **1h ~ 24h** | +2 | <10min（刚创建，rug风险） |
| 流动性(pool/mcap) | **15% ~ 30%** | +2 (15-30%), +1 (>30%) | <12%（无法退出） |
| 信号时涨幅 | <500% | — | >500%（已经拉过了） |

### 为什么这些阈值

- **MCAP<50K**: 太小，深度不够，买不进卖不出
- **MCAP>300K**: 太大，向上空间有限（平均最高才+27%）
- **年龄<1h**: 刚创建的币，rug/砸盘概率高
- **年龄>24h**: 老币，新鲜度下降，平均涨幅只有+46%
- **流动性<12%**: 池子太浅，大单会砸穿
- **涨幅>500%**: 信号发出时已经涨完了，接盘侠

## 第三层：量能确认

### 这是最重要的一层

**K线周期：5m（5分钟K线）**，因为策略所有历史数据都基于5m粒度。

### post/pre 量比定义

```
         信号触发时刻
             ↓
  ┌─────┬─────┬─────┬──┬──┬──┬──┬──┐
  │ -10 │ -5  │  0  │+5│10│15│20│25│  ← 相对于信号的时间（分钟）
  ├─────┼─────┼─────┼──┼──┼──┼──┼──┤
  │ K-2 │ K-1 │Ksig │K1│K2│K3│K4│K5│  ← 5m K线
  └─────┴─────┴─────┴──┴──┴──┴──┴──┘
          ← pre →       ← post →

pre  = 信号前5根K线（K-4 ~ K-0）的平均成交量
post = 信号后5根K线（K+1 ~ K+5）的平均成交量

post/pre = post ÷ pre
```

### 操作步骤

1. 信号触发后，**等待 15~25 分钟**（让5根K线走完）
2. 取出信号前5根K线的volume，算平均 → `pre_avg`
3. 取出信号后5根K线的volume，算平均 → `post_avg`
4. 计算 `ratio = post_avg / pre_avg`

### 判断

| post/pre | 含义 | 操作 |
|----------|------|------|
| **> 1.2** | 放量，真实需求进场 | ✅ 通过，正常仓位 |
| 0.8 ~ 1.2 | 量能持平 | ⚠️ 通过但减半仓位(5%) |
| **< 0.8** | 缩量，死猫反弹 | ❌ **丢弃，不交易** |

### 历史验证

```
胜者组 post/pre = 1.32   → 信号后放量32%，真实买盘推动价格
败者组 post/pre = 0.69   → 信号后缩量31%，拉高即出货

量比<0.8 时，75%的概率是 pump_crash 路径（先拉高再崩盘）
量比>1.2 时，60%的概率是 V_reversal 路径（先跌后涨）
```

## 第四层：入场规则

### 核心逻辑：等回撤再买

**永远不在信号价入场。** 60%的信号有先跌后涨的路径，先跌这段就是你的入场机会。

### 限价单挂法

```
限价买单价格 = 信号K线收盘价 × (1 - 回撤比例)
```

| 激进程度 | 回撤比例 | 挂单价格 | 预计成交率 |
|----------|---------|---------|-----------|
| 激进 | **35%** | 信号价 × 0.65 | 更高（回撤浅，容易触发） |
| **均衡（推荐）** | **40%** | 信号价 × 0.60 | 适中 |
| 保守 | 45% | 信号价 × 0.55 | 更低（回撤深，不一定到） |

### 入场时间限制

- **最大等待时间：8小时**
- 8小时内未成交 → 取消限价单
- 原因：信号时效性，超过8小时信号失效

### 不追高的原因

| 入场方式 | 交易数 | 平均收益 | 胜率 |
|----------|--------|---------|------|
| 信号价直接买入 | 150 | +26.9% | 44% |
| 等-40%回撤入场 | 7 | **+145.3%** | **100%** |

## 第五层：风控

### 成交后立即设置

| 条件 | 数值 | 含义 |
|------|------|------|
| **硬止损** | **-25%**（从入场价） | 跌25%无条件卖出 |
| **移动止盈激活** | **+50%**（从入场价） | 涨50%后启动追踪 |
| **移动止盈距离** | **15%**（从最高点） | 从最高点回落15%卖出 |
| **保本保护** | 入场价±5% | 到过+50%后回落，到入场价就出 |

### 硬止损计算

```
入场价 = $0.0000674
止损价 = $0.0000674 × 0.75 = $0.0000506
```

### 移动止盈计算

```
入场价 = $0.0000674
激活价 = $0.0000674 × 1.50 = $0.000101  ← 涨到这个价，启动移动止盈

如果最高涨到 $0.000339：
  止盈价 = $0.000339 × 0.85 = $0.000288  ← 从高点回落15%卖出
```

### 保本保护

```
如果到过 +50% 以上，然后跌回到入场价的 95%~105%：
  → 无条件卖出，保本出局
```

## 第六层：仓位管理

### 规则

| 参数 | 值 | 说明 |
|------|-----|------|
| 单笔仓位 | **5%~10%** 总资金 | meme币波动大，小仓位 |
| 最大并发持仓 | **3~5个** | 分散风险 |
| 日最大亏损 | **15%** 总资金 | 触发后停止当日交易 |

### 凯利公式参考

```
平均胜率 90%+，平均赔率 1.3:1
→ 理论最优仓位约 15-20%
→ 由于meme币流动性风险，折扣至 5-10%
```

## 第七层：止盈

### 分批止盈规则

| 仓位比例 | 止盈条件 | 逻辑 |
|----------|---------|------|
| **50%** | 涨到 +50% | 先锁定一半利润，落袋为安 |
| **30%** | 涨到 +100% | 让一部分利润跑 |
| **20%** | 移动止盈(15%) | 最后的仓位让趋势决定 |

### 示例

```
入场价 = $0.0000674
总仓位 = $1000（假设总资金$10000）

涨到 +50%（$0.000101）：
  → 卖出 50% × $1000 = $500 的仓位
  → 收回 $750，锁定 $250 利润

涨到 +100%（$0.000135）：
  → 卖出 30% × $1000 = $300 的仓位
  → 收回 $600，锁定额外 $300 利润

剩下 20%（$200）继续跑：
  → 最高涨到 $0.000339（+403%）
  → 移动止盈在 $0.000288（高点的85%）
  → 卖出剩余仓位，收回 $854
  → 剩余利润 $654

总收回：$750 + $600 + $854 = $2204
总利润：$1204（+120% 总仓位）
```

### 为什么不一次性全出

回测数据显示：V_moon 类标的平均最大涨幅 +305%，但平均当前收益仅 +138%。大量利润在高点蒸发。分批止盈在锁定利润的同时给一部分仓位飘的空间。

## 完整执行流程

### 时间线

```
T+0     收到信号推送
        → 第一层：查 signal_type ✓
        → 第二层：查 MCAP/Age/Liq ✓
        
T+0~5   什么都不做，等K线走
        
T+15~25 第三层：收集5根post信号K线，计算量比
        → post/pre > 1.2 ✓
        
T+25    第四层：挂限价单 @ -40% DD
        → 同时设好止损-25%、移动止盈激活+50%、止盈分三批
        
T+?     等待成交（最长8小时）
        → 如成交 → 第五层+第六层+第七层自动生效
        → 如8小时未成交 → 取消订单
```

### 检查清单

```
收到信号时：
□ 1. 信号类型是 abnormal / new_revival / quiet_runup？
□ 2. MCAP 50K-300K？
□ 3. 年龄 1h-24h？
□ 4. 流动性 15%-30%？
□ 5. 信号时涨幅 <500%？

等待15-25分钟后：
□ 6. post/pre 量比 >1.2？（<0.8 直接丢弃）

量能通过后：
□ 7. 挂限价单 @信号价×0.60~0.65
□ 8. 设止损 @入场价×0.75
□ 9. 设移动止盈激活@+50%，回撤15%
□ 10. 设保本保护
□ 11. 设止盈分批：50%@+50%, 30%@+100%, 20%移动
□ 12. 确认仓位≤10%，并发≤5

成交后：
□ 13. 每天检查一次持仓，检查移动止盈是否触发
□ 14. 日亏损15% → 当日停止交易
```

## 常见问题

### Q: 为什么用5m K线而不是15m或1h？

5m是在信号时效性和噪音之间的最佳平衡。1m噪音太大（假信号多），15m反应太慢（等你确认完，价格已经拉上去了）。所有历史回测基于5m。

### Q: post/pre量比在交易中怎么算？

```python
# 伪代码
pre_bars = klines[signal_idx-5:signal_idx]   # 信号前5根K线
post_bars = klines[signal_idx+1:signal_idx+6] # 信号后5根K线

pre_avg = sum(b.volume for b in pre_bars) / 5
post_avg = sum(b.volume for b in post_bars) / 5

ratio = post_avg / pre_avg

if ratio > 1.2:
    enter()  # 放量 → 入场
elif ratio < 0.8:
    skip()   # 缩量 → 丢弃
else:
    enter_small()  # 持平 → 减半仓位
```

### Q: 如果信号后价格一直在涨，永远等不到-40%回撤怎么办？

那就不交易这笔。回测数据中，追高买入的平均收益只有+27%（44%胜率），远低于等回撤的+130%（100%胜率）。错过一笔不可惜，买错一笔才亏钱。

### Q: 价格跌到-35%入场后继续跌怎么办？

硬止损在-25%（从入场价算）。如果你在-35%入场，相当于从信号价已跌了-35%，再跌-25%（入场价）才触发止损。从信号价算，止损在约-51%的位置。回测中-35%入场的止损触发率极低。

### Q: 量比在0.8-1.2之间怎么办？

量能持平说明方向不明确。减半仓位（5%而不是10%），并且对后续K线保持警惕。如果接下来的K线继续缩量，考虑取消限价单。

### Q: 已经有很多持仓（3-5个），又来新信号怎么办？

如果所有持仓都在盈利，且新信号评分>当前持仓 → 可以考虑换仓。
如果已有持仓在亏损 → 不新增，等现有持仓先解决。
日亏损达15% → 当日停止一切交易。

### Q: 什么时段信号最好？

晚间19:00-22:00 CST 和清晨07:00 CST 信号质量最高。凌晨01:00和下午16:00信号质量最差。但时段不是硬性过滤器——高质量信号在任何时段都可能出现。

### Q: 这套策略多久能产生一笔交易？

4天150个CA中，DD40%+全过滤产生了5-8笔交易，平均每天约1.5-2笔。这不是高频策略，是精选策略。
